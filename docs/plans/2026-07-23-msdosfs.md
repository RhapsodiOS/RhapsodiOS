# msdosfs (FAT12/16/32) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add full R/W FAT12/FAT16/FAT32 + VFAT support to the RhapsodiOS kernel, plus `/sbin/mount_msdos`, `/sbin/newfs_msdos`, `/sbin/fsck_msdos`, and `/usr/filesystems/msdos.fs/msdos.util` with autodiskmount wiring.

**Tech Stack:** Darwin 0.3 / Rhapsody BSD VFS, FreeBSD 4.11 msdosfs + sbin tools, Project Builder / `pb_makefiles`, autodiskmount Objective-C.

**Spec:** `docs/specs/2026-07-23-msdosfs-design.md`

**Upstream pin:** `https://raw.githubusercontent.com/freebsd/freebsd-src/release/4.11.0/`

## Critical FreeBSD ŌåÆ Darwin adaptations

These apply to **every** kernel `.c` import. Do them in Task 2 before expecting a link.

1. **No `VFS_SET` / `VNODEOP_SET` / `MALLOC_DECLARE`.** Darwin registers statically in `vfs_conf.c`.
2. **Export symbol names must match `vfs_conf.c`:**
   - `struct vfsops msdos_vfsops` (FreeBSD has `static struct vfsops msdosfs_vfsops`)
   - `struct vnodeopv_desc msdosfs_vnodeop_opv_desc` (remove `static`)
3. **`struct vfsops` field order differs.** Darwin (`mount.h`) is:

   ```c
   mount, start, unmount, root, quotactl, statfs, sync,
   vget, fhtovp, vptofh, init, sysctl
   ```

   FreeBSD 4.11 msdosfs uses `checkexp`, `uninit`, `extattrctl` and `vfs_std*` helpers that **do not exist** here. Rewrite the vector like `cd9660_vfsops` (see Task 2).
4. **`vfs_fhtovp` Darwin signature** includes `struct mbuf *nam`, `int *exflagsp`, `struct ucred **credanonp` ŌĆö fold FreeBSD `msdosfs_checkexp` logic into Darwin `msdosfs_fhtovp` or ignore export nam when `nam == NULL` (local mounts).
5. **Do not invent `vfs_stdstart` / `vfs_stdquotactl` / `vfs_stdvget`.** Provide local stubs returning 0 / `EOPNOTSUPP` as cd9660 does.
6. **Include style:** Prefer quoted local headers (`"denode.h"`, `"msdosfsmount.h"`). Use existing Darwin `<sys/ŌĆ”>`, `<vfs/ŌĆ”>` paths like cd9660.
7. **`MALLOC(M_MSDOSFSMNT)` etc.** already numbered in Darwin ŌĆö keep FreeBSD allocator type names; do not add new `MALLOC_DEFINE`.

### Task 9: `msdos.util` (WSM / autodiskmount)

**Files:**

- [ ] **Step 1: Implement probe (`FSUC_PROBE` / `-p`)**

Read sector 0; recognize FAT via BPB:
- `bsBytesPerSec` power-of-two 512..4096
- `bsFATs >= 1`
- FAT32: `bpbRootClust` path / `bsFileSysType` `"FAT32   "` or FAT12/16 `"FAT12   "` / `"FAT16   "` when present
- On success return `FSUR_RECOGNIZED` (-1); else `FSUR_UNRECOGNIZED` (-2)

Write volume label to the utilŌĆÖs label file mechanism the same way `cd9660.util` / `hfs.util` do (`DoFileSystemFile` pattern).
