# rhap-build CLI Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Replace `ninja/genninja` and the apk shell helpers with one multi-file C binary `ninja/rhap-build` that generates `build.ninja`, can run `samu`, and packages/indexes `.apk` files in-process.

**Architecture:** Split into `rhap-build.c` (CLI + `build` + `publish`), `generate.c` (former `genninja` body), `mkapk.c`, `index.c`, and shared `util.c`. Keep `buildproj.sh` as the ninja wrapper; point it at `rhap-build mkapk`. Delete `mkapk.sh`, `index-apk-repo.sh`, `publish-apk-repo.sh`, and `genninja.c`.

**Tech Stack:** POSIX C (`cc -Wall`), host `tar`/`gzip`, vendored `samu`, existing `ninja/tests/*.sh` fixtures.

**Spec:** [docs/superpowers/specs/2026-07-19-rhap-build-design.md](../specs/2026-07-19-rhap-build-design.md)

- [ ] **Step 3: Add `mkapk.h` / `mkapk.c`**

```c
/* mkapk.h */
#ifndef RHAP_MKAPK_H
#define RHAP_MKAPK_H
int mkapk(const char *pkginfo, const char *root, const char *out);
#endif
```

Behavior (match `mkapk.sh`):

1. Validate `pkginfo` is a file, `root` is a directory, `out` non-empty â†’ else fprintf usage, return 2.
2. `mkdtemp` under `$TMPDIR` or `/tmp/mkapk.XXXXXX`.
3. `cp`/`read`+`write` PKGINFO to `$tmp/.PKGINFO`.
4. `system`/`popen` equivalent of `(cd root && tar cf - .) | (cd tmp && tar xf -)`.
5. Build member list with `.PKGINFO` first (use `opendir`/`readdir`, skip `.`/`..`).
6. If `gzip` on PATH: `tar cf - .PKGINFO â€¦ | gzip -n > out`; else `tar czf out .PKGINFO â€¦`.
7. `rm -rf` temp; return 0 on success, 1 on I/O failure.

Use `snprintf` for commands; escape is unnecessary if paths are trusted build paths (same as shell scripts). Prefer `execlp`/`fork`/`pipe` if comfortable; `system()` with carefully built command strings is acceptable and matches host assumptions.

```c
/* index.h */
#ifndef RHAP_INDEX_H
#define RHAP_INDEX_H
int index_apk_repo(const char *repo);
#endif
```

Logic port of `index-apk-repo.sh`:

- Not a directory â†’ stderr, return 1.
- Collect `repo/*.apk` via `opendir`; if none â†’ print skip, return 0.
- Resolve apk path: `APK` env if executable; else `$DSTROOT/sbin/apk`, `$TOOLROOT/sbin/apk`; else `"apk"` on PATH (`access(X_OK)` / search).
- Missing apk â†’ warn, return 0.
- Run `apk index file1.apk â€¦` with stdout piped to `gzip -n` writing `repo/APK_INDEX.gz.new`, then `rename` to `APK_INDEX.gz`.
- On pipeline failure return 1.

- [ ] **Step 1: Implement argv dispatch per spec**

Known subcommands: `generate`, `build`, `mkapk`, `index`, `publish`.

Algorithm:

1. Scan argv for first non-option that matches a subcommand (or treat no subcommand as bare generate+hint).
2. Parse generate options via `generate_parse_args` for bare / `generate` / `build`.
3. Behaviors:
   - **bare:** `generate_build_ninja`; on success print:

```text
rhap-build: wrote build.ninja (N projects)
rhap-build: ready. From the repo root run:
  ninja/samurai/samu buildworld
  ninja/samurai/samu buildkernel
Or: ninja/rhap-build build [target...]
```

   - **`generate`:** generate only (same stderr â€śwrote â€¦â€ť as today, no ready hint).
   - **`build`:** generate; then resolve samu:
     - `--samu PATH` consumed from args after `build`
     - else `SAMU` env
     - else `<repo>/ninja/samurai/samu` where repo is parent of directory containing this binary if path ends with `/ninja`, else cwd
     - Default target `buildworld` if no remaining args
     - `chdir(repo_root)`; `execv`/`spawn` samu with targets; return its status
   - **`publish`:** require `argc >= 5` and odd trailing count of pairs:

```c
/* for i = 3; i+1 < argc; i += 2 */
/* read pkgname/pkgver from pkginfo file (simple line scan) */
/* snprintf out = "%s/%s-%s.apk", repo, name, ver */
/* mkapk(pkginfo, stage, out) */
index_apk_repo(repo);
```
